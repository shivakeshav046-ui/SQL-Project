# AdventureWorks SQL Business Analysis

## 📌 Project Overview

This project analyzes the AdventureWorks database using SQL to solve real-world business problems. The analysis focuses on sales performance, customer behavior, product performance, and territory analysis by answering business questions through SQL queries.

## 🎯 Project Objective

- Analyze sales and customer data.
- Answer business questions using SQL.
- Generate business insights from the AdventureWorks database.

## 🛠️ Tools Used

- SQL Server
- SQL Server Management Studio (SSMS)

## 📂 Dataset

- **Dataset:** AdventureWorks
- **Database Type:** Relational Database
- **Source:** Microsoft AdventureWorks Sample Database

---

## 📋 Business Questions

**Q1.** Overview

**Q2.** Which product categories generate the highest revenue and profit?

**Q3.** Which product subcategories have high sales but low profit margins?

**Q4.** How has monthly revenue changed over time and where are the major growth or decline periods?

**Q5.** Which territories contribute the most revenue and profit?

**Q6.** Who are the top 10 customers by lifetime value (LTV)?

**Q7.** What percentage of total revenue comes from the top 20% of customers?

**Q8.** Which customer segments generate the highest average order value?

**Q9.** Which products generate the highest profit, not just revenue?

**Q10.** Which products has low sales in the top category

**Q11.** Which territories show the highest revenue growth rate year-over-year?

**Q12.** Are there territories with high order volume but low profitability?
